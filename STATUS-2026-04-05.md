## What Is This Project?

**The Pour List** — a crowd-sourced happy hour directory starting with Portland's Pearl District (97209), designed to scale to any city.

- **Name:** The Pour List (scalable beyond Portland)
- **GitHub:** xalanx200x
- **Repo:** to be created as `pourlist`

### Core Value Proposition
- Mobile-first PWA for walking around a neighborhood and finding happy hours nearby
- Users contribute photos of menus, not reviews
- No accounts required — device fingerprinting for spam prevention
- Third-party review links (Google/Yelp) — no own review system to moderate
- Progressive moderation: auto-flagging, community flagging, contributor trust levels

## Database Schema (Supabase)

File: `supabase-schema.sql` — **must be run manually in Supabase dashboard.**

### Tables

**venues**
| Column | Type | Notes |
|--------|------|-------|
| id | uuid | PK, auto-generated |
| name | text | venue name |
| address | text | street address |
| lat / lng | double precision | GPS coords |
| zip | text | defaults to 97209 |
| phone | text | optional |
| website | text | optional |
| type | text | bar, restaurant, lounge, etc. |
| status | text | unverified / verified / stale / closed |
| contributor_trust | text | new / trusted |
| last_verified | timestamptz | auto-updated on photo approval |
| photo_count | int | auto-incremented on photo approval |

**photos**
| Column | Type | Notes |
|--------|------|-------|
| id | uuid | PK |
| venue_id | uuid | FK → venues |
| url | text | Supabase Storage URL |
| uploader_device_hash | text | anonymous device ID |
| lat / lng | double precision | from photo EXIF or geolocation |
| status | text | pending / approved / rejected |
| flagged_count | int | for auto-moderation |
| moderation_confidence | double precision | AI moderation score |

**flags**
| Column | Type | Notes |
|--------|------|-------|
| id | uuid | PK |
| venue_id | uuid | FK → venues (nullable) |
| photo_id | uuid | FK → photos (nullable) |
| reason | text | flag reason text |
| device_hash | text | who flagged it |

### Auto-Triggers
- `update_venue_verified()` — when a photo is approved, bumps last_verified + increments photo_count
- `mark_stale_venues()` — marks venues as "stale" if no photos approved in 90 days

### Row Level Security
All tables have RLS enabled with `public read + insert + update` policies. No auth required.

## What Was Built

### Map + List View
- Mapbox map centered on Pearl District (45.5231, -122.6819)
- Venue markers colored by status (amber = unverified, yellow = stale)
- Click marker → venue card highlights + detail panel opens
- Split view on desktop: map left, venue list right
- Mobile: map full-screen, tap venue → switches to list + detail panel

### Venue Detail Panel
- Venue name, address, phone, website, type badge
- "Directions" button → opens Google Maps
- "⭐ on Google" button → opens Google Maps review page
- Photo upload button → captures photo, submits for review

### Add Venue Form
- For venues not yet in database
- Fields: name (required), address (required), phone, website, type (dropdown)
- Auto-reverse-geocodes from GPS if available
- Sets zip to 97209 by default (Pearl District scope for now)

### Device Fingerprinting
- Anonymous hash of userAgent + screen + timezone (no personal data)
- Used to track contributor trust and collapse spam from same device
- Used to group flags from the same device

### Geolocation
- Browser Geolocation API as fallback when photo lacks EXIF GPS
- Nominatim reverse geocoding for coordinates → address

## Key Design Decisions Made

| Decision | Rationale |
|----------|-----------|
| Mapbox over Google Maps | Free tier (50k loads/mo) vs Google's $200/mo credit (works but unnecessary) |
| Nominatim for geocoding | Free, no API key, sufficient for Pearl District scale |
| No accounts | Low barrier to entry; device hash is anonymous spam prevention, not surveillance |
| Link to Google/Yelp reviews | Zero moderation burden; third parties handle it |
| Pending → approved photo flow | Auto-moderation via AI + community flags; no pre-screening bottleneck |
| Bottom sheets on mobile | Native app feel; doesn't navigate away from map |
| 97209 only for now | Quality threshold before expanding to other neighborhoods |
