## What Was Built

### App Pages
- **`/`** — Home: map + list view, radius selector (0.25–25 mi), scan button
- **`/api/parse-menu`** — POST: sends photo URL to GPT-4o mini, returns extracted menu text
- **`/api/upload-photo`** — POST: uploads photo to Supabase Storage, returns URL
- **`/api/submit-menu`** — POST: saves menu text to venue record (creates venue if new)
- **`/api/check-duplicate`** — POST: checks if similar photo was recently uploaded

### Components
- `Map.tsx` — Mapbox map with venue markers, hover fix applied

- `MenuCapture.tsx` — Camera/gallery picker with GPS extraction

## Menu Scan Workflow (Complete)

```
User taps "Scan Happy Hour Menu"
    ↓
MenuCapture: Camera/gallery → extracts GPS from EXIF or browser
    ↓
handleCapture (page.tsx):
  1. Upload photo → Supabase Storage (returns URL)
  2. GPS lookup: find venue within 50m of coordinates
  3. Check duplicate via fingerprint
  4. If duplicate → show existing menu
  5. If new → call /api/parse-menu → GPT-4o mini extracts text
    ↓
MenuConfirm: Shows parsed text → user edits/confirms
    ↓
handleMenuConfirm: /api/submit-menu → saves menu_text to venue
    ↓
Photo discarded, menu text is permanent artifact
```

**Cost:** GPT-4o mini = $0.0003/photo. Effectively free at any reasonable scale.

## Design Decisions Made

| Decision | Rationale |
|---|---|
| Store parsed menu text, not photos | Photo is scanner input; text is the product. Photo deleted after parse. |
| Mapbox over Google Maps | Free tier (50k loads/mo) vs Google's $200/mo credit |
| Nominatim for geocoding | Free, no API key, sufficient for Pearl District |
| GPT-4o mini for parsing | $0.0003/photo — cheapest reliable option |
| No accounts | Anonymous device hash for spam prevention |
| Link to Google/Yelp | Zero moderation burden; third parties handle reviews |
| Bottom sheets on mobile | Native feel; doesn't navigate away from map |
| 97209 scope first | Quality threshold before expanding |
