## Venue Addition Flow (UPDATED 2026-04-17)

**Location-first UX — duplicates prevented at choice point:**

1. User taps **"Scan Menu"**
2. App checks **browser location** immediately (before photo)
3. Finds **nearby venues within 50m** → shows list to select
   - User selects venue → photo capture → GPS verification (50m check) → review
   - No venues nearby → "Add Happy Hour Location" → photo capture → GPS verification → review
4. If pre-selected venue GPS verification fails → user can confirm venue name or pick different one

**Duplicate prevention:** Users snap to existing venues before contributing. Only new venues go through creation flow.

**GPS + EXIF matching (for new venues — no addresses required):**

1. User takes photo of menu
2. Photo EXIF extracted → GPS coordinates
3. Browser GPS via `navigator.geolocation`
4. **Both must be within 10m** to prove user was at venue (reconciled from ~50m)
5. If EXIF missing or GPS mismatch → submission rejected
6. Device hash tracked for reputation

**No fallback for missing EXIF or GPS.** Phones without geotagging capability cannot contribute.

## Venue Removal/Moderation Flow (FINALIZED 2026-04-17)

**Core rules — all finalized after Cato Phase 3 QA:**

| Rule | Value |
|------|-------|
| Flag GPS required | YES — 10m Haversine |
| Confirm GPS required | YES — 10m Haversine |
| Device flags same venue twice | BLOCKED — permanent, ever |
| Can re-flag after decay to 0 | BLOCKED — once you've flagged, never again |
| Flags per device per venue | 1 per day (hidden in UI) |
| Flags before submissions count | 1 successful submission required |
| Flag decay | 1 flag removed per month |
| Flagger can confirm same venue | NO |
| Flagger can reopen own close | NO |
| Flags on venue reopen | All deleted |
| Menu committed to DB | All flags on venue cleared |
| Trusted device flag weight | 2x toward N threshold |
| >50% discount rule | DROPPED |

**Flag thresholds:**
- N=2 distinct flagging devices → venue status: `'stale'` (hidden from main map)
- N=4 distinct flagging devices → venue status: `'closed'` (fully hidden)

**What counts as a "distinct flagging device":**
- One device hash = one device = one flag per venue (per day, but effectively once)
- Device cannot flag the same venue twice — ever, even after decay
- All flags from the same device toward the same venue count as 1 toward N

**Confirm ("I'm here, it's still valid"):**
- Requires GPS proof — same 10m Haversine check
- Cancels all existing flags on the venue
- A device that has flagged a venue CANNOT confirm that same venue
- A device that flagged a venue CANNOT confirm it after it reopens

**Menu submission clears flags:**
- When a menu submission is committed to the DB (passed spam checks + rate limit)
- All flags on that venue are deleted
- This is a strong trust signal: the venue is real enough to attract submissions

**Flag decay:**
- 1 flag removed per month per venue (automatic)
- If flags reach 0, the device that placed the flag CANNOT flag that venue again
- Flags do not "freeze" decay — if a venue has 1 flag and it's 2 months old, that flag is gone

**Trusted device:**
- After 10 successful submissions → `'trusted'`
- Trusted flags count 2x toward N threshold
- A trusted device can still only flag a venue once

**Race conditions (flag vs confirm):** Engineering implementation — use DB row-level locking. Both writes are accepted; final state reflects whichever completed last. Flag to builder.

**Soft delete only.** Closed venues can be reopened by any device that hasn't flagged them. No hard deletes.

## Design Decisions

| Decision | Rationale |
|---|---|
| GPS + EXIF for proof of presence | No fake submissions from remote; requires physical presence |
| No fallback for missing EXIF/GPS | Simplicity; older phones excluded |
| N=2 stale, N=4 closed | Low enough for responsiveness, high enough to prevent casual abuse |
| Soft delete only | Audit trail preserved; venues can be restored |
| 10 submissions for trusted | Meaningful threshold without being exclusionary |
| Google Places verification | Prevents fake business spam; establishes ground truth |
| address_backup preserved | Eventually replace with Places API data; backup during transition |
